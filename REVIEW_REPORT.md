### main.py
1. **密码摘要算法不安全（安全漏洞）**
   - 在 `_build_login_password` 中使用了 `hashlib.md5` 对密码做摘要后传输。MD5 已不再满足现代安全要求，存在碰撞风险，不适合作为密码相关场景的安全保护手段。
   - 在 Python 3.10 异步环境下，虽不影响协程调度，但该实现会降低凭证保护强度。

2. **默认使用明文 HTTP 传输凭据（安全漏洞）**
   - `_get_base_url` 的回退地址为 `http://127.0.0.1:6185`，并且登录流程直接将账号与（MD5 后）口令提交到该地址；若部署并非严格本机回环/可信链路，存在被窃听与中间人攻击风险。
   - 建议在配置校验中明确限制生产环境使用 HTTPS（或至少在非回环地址下强制 HTTPS）。

3. **网络错误分类粒度不足导致诊断信息丢失（可维护性/潜在缺陷）**
   - `_refresh_help_cache` 对 `PermissionError` 统一返回“登录状态失效”，但该异常也用于 `login_unauthorized` 与 `token_unauthorized` 两种不同阶段，用户侧无法区分“账号密码错误/权限拒绝”与“token 过期/失效重试失败”，不利于快速定位问题。
   - 建议按阶段给出更精确错误文案，或引入自定义异常类型区分登录失败与 token 失效。
